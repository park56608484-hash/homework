### 목표 : 리눅스 명령어 *top*, *ps*, *jobs*, *kill* 의 역할과 사용법

---

#### 1. *`top`*

+ ##### 역할(Role) 
   > 리눅스 시스템의 실시간 상태(프로세스, CPU, 메모리 사용량)을 보여주는 모니터링 도구
   + 실행 중인 **프로세스 목록**을 실시간으로 확인 
   + 각 프로세스의 **CPU 메모리 사용량**을 모니터링
   + 시스템 전체 **부하(load) 상태**를 한눈에 확인

+ ##### 사용 목적(Purpose)
  > "**지금 누가 자원을 많이 쓰고 있는지**"를 알고 싶을 때 사용하는 명령어
   + 서버나 프로그램이 **느려졌을 때 원인(문제 프로세스)** 을 찾을 때
   + **CPU, 메모리를 과도하게 사용하는 프로세스**를 골라 확인, 종료할 때
   + 시스템이 **과부하인지 주기적으로 점검**할 때

+ ##### 기본 조작
  + `q` : **top 종료**
  + `P` : **CPU 사용량 기준으로 정렬**
  + `M` : **메모리 사용량 기준으로 정렬**
  + `1` : **각 CPU 코어별 사용량 표시/숨기기**

#### 2. *`ps`*

+ ##### 역할(Role)
   > `ps`는 **현재 실행 중인 프로세스 정보를 "한 번만" 출력해 주는 명령어**
   + 실행 중인 **프로세스 목록**을 출력
   + 프로세스를 **누가 어떤 옵션으로 실행했는지** 확인가능.
   + 특정 조건(PID, 사용자 등)에 맞는 **프로세스만 골라서 조회**

+ ##### 사용 목적(Purpose)
   >"**지금 어떤 프로세스가 떠 있는지, 그 상세 정보가 궁금할 때**" 사용하는 명령어
   + 백그라운드로 띄운 프로그램이 **잘 떠 있는지 확인**할 때
   + 특정 프로세스의 **PID를 알아내서 `kill`로 종료**할 때
   + 어떤 계정이 어떤 프로세스를 실행 중인지 **사용자별 프로세스**를 보고 싶을 때

+ ##### 기본 조작
   + `ps` : **현재 쉘에서 실행 중인 내 프로스세만 출력**
   + `ps aux` : **시스템에 떠 있는 모든 프로세스를 상세하게 출력**
   + `ps aux | grep 이름` : **특정 이름을 포함한 프로세스만 검색해서 보기**
   + `ps -u USER` : **특정 사용자(USER)가 실행 중인 프로세스만 보기**
   + `PS -p PID` : **특정 PID 하나에 대한 정보만 보기**

#### 3. *`jobs`*

+ ##### 역할(Role)
   > `jobs`는 **현재 쉘에서 백그라운드로 실행 중이거나 일시 중지된 작업 목록을 보여주는 명령어**
   + 현재 쉘에서 실행한 **백그라운드 작업 리스트**를 보여줌.
   + `Ctrl + Z`로 **일시 중지시킨 작업 상태**를 확인가능.
   + `fg`, `bg`, `kill` 명령어와 함께 **작업 제어**를 할 때 사용
 
+ ##### 사용 목적(Purpose)     
  >"**지금 이 쉘에서 어떤 작업들이 뒤에서 돌아가고 멈춰있는지** 확인하고 싶을 때" 쓰는 명령어
   + `&`를 붙여 **백그라운드로 돌려놓은 작업들을 확인**할 때
   + `Ctrl + Z`로 잠깐 멈춘 작업이 **몇 번 작업 번호**인지 알고 싶을 때
   + `fg`, `bg`, `kill %번호`로 조작하기 전에 **현재 작업 상태를 체크**할 때

+ ##### 기본 조작
   + `jobs` : **현재 쉘의 백그라운드/일시 중지 작업 목록 보기**
   + `jobs -l` : **작업 번호 + PID(프로세스 ID)까지 함께 보기**
   + `jobs -r` : **실행 중인 작업만 보기**
   + `jobs -s` : **일시 중지된 작업만 보기**

---

#### 4. *`kill`*
+ ##### 역할(Role)
   > `kill`은 **지정한 프로세스에 종료 신호를 보내는 명령어**
   + 실행 중인 **프로세스를 정상 종료**
   + 응답하지 않는 프로세스를 **강제 종료**
   + 단순 종료뿐 아니라 **다양한 시그널(일시 정지,재개 등)** 도 보낼 수 있음.

+ ##### 사용 목적(Purpose)
   >"**지금 돌아가는 특정 프로세스를 끝내거나, 신호를 보내서 제어하고싶을 때**" 사용하는 명령어
   + CPU, 메모리를 너무 많이 쓰는 **문제 프로세스를 종료**할 때
   + 멈춰버린 프로그램이 **반응하지 않을 때 강제 종료**할 때
   + 데몬/서버 프로세스에 **재시작 신호를 보내고 싶을 때**
  
+ ##### 기본 조작
     + `kill PID` : **지정한 PID 프로세스에 기본 종료 신호 보내기**

#### 마무리 정리
| 명령어 | 역할 |
|---|---|
|top | 실행중인 프로세스와 CPU, 메모리 사용량을 **실시간으로 모니터링**|
|ps | 현재 실행 중인 프로세스 목록을 **한 번 출력**
|jobs | 현재 쉘의 **백그라운드/일시중지 작업 목록**을 확인
|kill| 특정 프로세스에 신호를 보내 **종료, 중지, 재개를 제어**
